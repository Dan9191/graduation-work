# Концепция проекта.
Веб-сервис для хранения данных и поиска с использованием RAG технологий 

## Предпосылки.
* Потребность в собственной базе знаний
* Нужная информация в интернете может находиться на разных ресурсах и ее не просто найти
* Требуется хранить специфичные или узкоспециализированные данные

## Задача.
Создать ресурс, который позволит:
* Хранить данные в виде статей
* Структурировать данные по меткам и секциям
* Выдавать структурированный и развернутый ответ, основанный на нескольких статьях

## Проблема.
В повседневной жизни бывает такое, что при поиске информации мы вынуждены открывать множество ресурсов,
при этом полученный результат может быть неудовлетворительным.
Информация может быть разрозненной, неактуальной или вовсе ложной, а повлиять мы на это не можем.

## Возможности, которые дает проект.
Структурированное создание и хранение важной для пользователя информации с возможностью получения ответа,
основанного на разрозненных данных.

## Цель по SMART.
* **Specific:**
Создать веб сервис для хранения и поиска информации.
* **Measurable:**
Реализовано долгосрочное хранение информации. При запросе, веб сервис выдает релевантный ответ за не более, чем 30 секунд
* **Achievable:**
Использование готовых моделей для векторизации и формирования ответов сокращает время разработки
и убирает требование к высокой личной экспертизе.
* **Relevant:**
Актуально для пользователей, которые стремятся к созданию своей базы знаний для хранения важной информации.
* **Time-bound:**
3 месяца до MVP и 6 до готового продукта.

## Ожидаемый эффект.
* Сокращение времени поиска информации в интернете
* Рост в эффективности усвоения информации
* Единое пространство, в котором хранится нужная информация

## Критерий успеха.
Веб сервис способен хранить статьи в количестве больше 1000 штук и выдавать релевантный ответ за 30 секунд
